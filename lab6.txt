4. You can use the genrsaLinks to an external site. command to generate a key pair.  However, since it was deprecated, we are going to use the genpkeyLinks to an external site. command instead. You can check the command usage by typing openssl genpkey -help.  To generate an RSA private key using default parameters and save it to the key.pem file:

openssl genpkey -algorithm RSA -out key.pem
However, we will use the following command, keep note of the password you set (here it is webapps):

openssl genpkey -algorithm RSA -out key.pem -pkeyopt rsa_keygen_bits:2048 \
-aes-128-cbc -pass pass:webapps
which encrypts the key with a 128 bits AES algorithm, protects the key with a passphrase, and the output key has 2048 bits. You can check only the header part of the private key with the command:

openssl rsa -in key.pem -text -noout
5. Now, extract the public key from the private key with the command:

openssl rsa -in key.pem -pubout > public.pem
 You can check the public key details with the command:

 openssl rsa -in public.pem -pubin -text -noout
6. Now, you need to generate a Certificate Signing Request (CSR), one of the first steps towards getting an SSL/TLS certificate. A CSR includes a public key and extra information, e.g., common name, organization, and country, inserted into the certificate. Remember to provide accurate information if the certificate is being bought from a CA. To create a CSR after providing all of the necessary information along with a private key, you can use the following command:

openssl req -newkey rsa:2048 -nodes -keyout domain_name.key -out domain_name.csr
Please note that it is created a 2048-bit RSA key that is not protected by a passphrase. You should replace domain_name with the domain name of your site, e.g. localhost or webapps. Once the CSR file is created, you can send it to a CA and request a certificate. If the private key already exists, you should proceed as follows:

openssl req -key domain_name.key -new -out domain_name.csr
It is also possible to get the extra security of an SSL certificate by signing it yourself with your private key, also known as a self-signed certificate. A self-signed certificate is as effective as a CA-signed certificate. However, due to the lack of a TTP, their use is recommended within an organization. You may also receive a warning from your browser that they are not secure. 

7. You can generate a 2048-bit private key and a self-signed certificate to use HTTPS to secure a web server with the command

openssl req -x509 -newkey rsa:2048 -keyout domain_name.key -out domain_name.crt -sha256 -days 365
8. You can view the certificate content with the following command

openssl x509 -text -noout -in domain_name.crt
